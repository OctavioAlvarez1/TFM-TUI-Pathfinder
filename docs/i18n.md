# Internacionalizacion (i18n) — TUI Pathfinder

Pathfinder soporta dos idiomas: **espanol (ES)** e **ingles (EN)**. El sistema esta implementado con un contexto React personalizado — sin dependencias externas de i18n.

---

## Arquitectura del sistema

```
src/
├── i18n/
│   └── translations.ts          # Objeto typed as const — todas las claves x idioma
└── context/
    └── LanguageContext.tsx       # Proveedor + hook useLanguage()
```

---

## LanguageContext

El contexto expone tres valores:

```typescript
const { lang, setLang, t } = useLanguage()
// lang    : 'es' | 'en'
// setLang : (lang: Lang) => void
// t       : (key: TranslationKey) => string
```

El idioma activo se persiste en `localStorage` con la clave `pathfinder_lang`. Al recargar la pagina se recupera el idioma seleccionado.

---

## Archivo de traducciones

`src/i18n/translations.ts` contiene un objeto declarado `as const` con dos bloques: `es` y `en`. Cada bloque es un record plano de clave → string.

```typescript
export const translations = {
  es: {
    'nav.map':              'Mapa interactivo',
    'nav.accessibility':    'Accesibilidad',
    // ...
  },
  en: {
    'nav.map':              'Interactive map',
    'nav.accessibility':    'Accessibility',
    // ...
  },
} as const
```

El tipo `TranslationKey` se deriva automaticamente:

```typescript
export type TranslationKey = keyof typeof translations.es
// TypeScript falla en tiempo de compilacion si se usa una clave que no existe
```

## Convenio de nombres de clave

Las claves siguen el patron `<prefijo>.<subprefijo>.<nombre>`:

| Prefijo | Vista / area |
|---|---|
| `nav.*` | Sidebar — etiquetas de navegacion |
| `topbar.*` | TopBar — periodo, avatar |
| `mob.*` | MobilityView |
| `acc.*` | AccessibilityView |
| `routes.*` | TouristRoutesView |
| `ai.*` | AIRecsView |
| `analytics.*` | AnalyticsView |
| `reports.*` | ReportsView |
| `imap.*` | InteractiveMapView |
| `common.*` | Textos reutilizables |

## Como anadir una clave nueva

**Paso 1** — Agregar la clave a `translations.ts` en **ambos idiomas**:

```typescript
// en el bloque es:
'mob.new_label': 'Etiqueta nueva',

// en el bloque en:
'mob.new_label': 'New label',
```

> Si se agrega solo en un idioma, TypeScript falla al compilar con un error de tipo en `TranslationKey`.

**Paso 2** — Usar la clave en el componente:

```typescript
const { t } = useLanguage()
// ...
<Typography>{t('mob.new_label')}</Typography>
```

## Patrones especiales

### Arrays via pipe-delimited strings

Para listas de valores (meses, nombres, recomendaciones) que deben traducirse, se guarda un unico string con `|` como separador y se divide en el componente:

```typescript
// En translations.ts:
'analytics.months': 'Ene,Feb,Mar,Abr,May,Jun,Jul,Ago,Sep,Oct,Nov,Dic',
// EN:
'analytics.months': 'Jan,Feb,Mar,Apr,May,Jun,Jul,Aug,Sep,Oct,Nov,Dec',

// En el componente:
const MONTH_LABELS = useMemo(() => t('analytics.months').split(','), [t])
```

Para arrays separados por `|`:

```typescript
// En translations.ts:
'imap.zone.hot_names': 'Centro hist.|Puerto|Paseo Maritimo|Plaza Mayor',
// EN:
'imap.zone.hot_names': 'Historic Centre|Harbour|Seafront|Main Square',

// En el componente:
const ZONE_HOT = t('imap.zone.hot_names').split('|')
```

### Dynamic key cast

Cuando la clave se construye dinamicamente (con template literal), TypeScript no puede inferirla como `TranslationKey`. Usar el cast explicito:

```typescript
// Con sufijo de variable:
t(`analytics.range.${key}` as Parameters<typeof t>[0])

// Con indice numerico:
([1, 2, 3] as const).map(n =>
  t(`analytics.trend.${n}` as Parameters<typeof t>[0])
)
```

### Datos que cambian con el idioma dentro de useMemo

Cuando un `useMemo` depende de valores traducidos, incluir `t` en el array de dependencias:

```typescript
const MONTH_LABELS = useMemo(() => t('analytics.months').split(','), [t])
// t cambia al cambiar el idioma -> MONTH_LABELS se recalcula
```

## Datos estaticos con traduccion en componente

Las estructuras de datos con strings (objetos de configuracion, arrays de labels) **no pueden usar `t()` a nivel de modulo** porque `t` es un hook. Patron correcto:

```typescript
// INCORRECTO — t() no esta disponible fuera del componente
const MODE_META = {
  concentracion: { label: 'Concentracion' },   // string en duro
}

// CORRECTO — colores en modulo, labels traducidas en componente
const MODE_COLORS = {
  concentracion: { highColor: '#EF4444', lowColor: '#10B981' },
}

// Dentro del componente:
const { t } = useLanguage()
const MODE_META_T = {
  concentracion: { ...MODE_COLORS.concentracion, label: t('imap.meta.conc') },
}
```

## Selector de idioma (TopBar)

El selector esta implementado en `TopBar.tsx` junto a la campanita de notificaciones. Usa imagenes de banderas desde `flagcdn.com` en lugar de emojis (los emojis de banderas regionales no se renderizan en Windows):

```typescript
const FLAG_SRC: Record<Lang, string> = {
  es: 'https://flagcdn.com/w40/es.png',
  en: 'https://flagcdn.com/w40/us.png',
}

// Renderizado:
<Box
  component="img"
  src={FLAG_SRC[l]}
  alt={l}
  sx={{ width: 22, opacity: lang === l ? 1 : 0.55 }}
/>
```

La bandera activa aparece a opacidad completa con fondo destacado; la inactiva a 55% de opacidad.

## Anadir un nuevo idioma

Para anadir un tercer idioma (ej: frances `fr`):

1. Agregar `'fr'` al tipo `Lang` en `LanguageContext.tsx`:
   ```typescript
   export type Lang = 'es' | 'en' | 'fr'
   ```

2. Anadir el bloque `fr` en `translations.ts` con todas las claves del bloque `es`.

3. Agregar la bandera en `TopBar.tsx`:
   ```typescript
   const FLAG_SRC: Record<Lang, string> = {
     es: 'https://flagcdn.com/w40/es.png',
     en: 'https://flagcdn.com/w40/us.png',
     fr: 'https://flagcdn.com/w40/fr.png',
   }
   ```

4. TypeScript indicara cualquier clave que falte en el nuevo bloque de idioma.
